# 📊 Sales Data Analysis (SQL Project)

## 📌 Project Overview
This project focuses on analyzing sales data using SQL to extract meaningful business insights. The analysis helps identify key trends, top-performing products, and customer behavior to support data-driven decision-making.

---

## 🛠 Tools & Technologies
- SQL (MySQL)
- MySQL Workbench

---

## 📂 Dataset
- Contains 10,000+ sales records
- Includes customer, product, region, and sales details

---

## 🔍 Key Analysis Performed

### 1. Overall Revenue
- Calculated total sales to understand overall business performance

### 2. Regional Performance
- Identified top-performing regions contributing to revenue

### 3. High-Value Customers
- Found top customers generating the highest sales

### 4. High-Revenue Products
- Identified best-selling products contributing to revenue

### 5. Category Performance
- Analyzed which product categories perform best

### 6. Seasonal Sales Trends
- Evaluated monthly sales trends to identify seasonality patterns

### 7. Customer Segmentation
- Analyzed sales based on different customer segments

### 8. Top Cities
- Identified cities generating the highest sales

---

## 📊 Key Insights
- Certain regions contribute significantly more to overall revenue
- A small group of customers drives a large portion of sales
- Specific products consistently perform better than others
- Sales show seasonal trends, indicating peak and low demand periods

---

## 🚀 How to Run
1. Import dataset into MySQL
2. Create database: `sales_project`
3. Run the SQL script (`sales_analysis.sql`)
4. Analyze results using MySQL Workbench

---

## 🎯 Conclusion
This project demonstrates the use of SQL for real-world data analysis, including data cleaning, aggregation, and business insight generation.
